Serialis – Resumen funcional
Módulos principales
  • Agenda: creación/edición de turnos, estados y recordatorios.
  • Historias clínicas: evolución, sesiones y adjuntos.
  • Pacientes: alta, datos de contacto, antecedentes.
  • Usuarios y roles: médicos, recepción, administración.
Tecnologías
  • Frontend: Bootstrap + JS (validaciones y UX responsiva).
  • Backend: PHP con control de sesiones y autenticación.
  • Datos: MySQL (tablas pacientes, turnos, historias, usuarios, etc.).
Buenas prácticas
  • Separación por roles/permisos y registro de acciones básicas.
  • Validación de formularios y manejo de estados.
  • Estructura preparada para mensajes/recordatorios.
  • Diseño responsive y componentes reutilizables.
Privacidad: el entorno contiene datos sensibles de salud. Las demos se realizan con datos ficticios o capturas anonimizadas.
